How to Format JSON Properly

Well-formatted JSON is easier to read, debug, and share. This guide covers the key rules for formatting JSON, including indentation, structure, and common best practices.

Why formatting matters

JSON is often generated in a compact format by machines, but human readers benefit from indentation and line breaks. Proper formatting helps you identify nested objects, arrays, and error locations quickly.

Formatting best practices

  • Use consistent indentation (usually 2 or 4 spaces)
  • Keep keys in double quotes
  • Break objects and arrays across lines for readability
  • Place commas at the end of lines, not the beginning
  • Use online tools to validate and beautify JSON before sharing

Example

{
  "name": "Jane Doe",
  "email": "jane@example.com",
  "roles": [
    "admin",
    "editor"
  ],
  "settings": {
    "theme": "dark",
    "notifications": true
  }
}
